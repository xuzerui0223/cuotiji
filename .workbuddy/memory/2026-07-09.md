- 根因：原举一反三只是把用户已有错题按题型标签重做，没有新题来源。用户确认需求后定为：内置题库 + 按「学科+知识点」匹配 + 答错新题自动入本。
- 新增 `kp`(知识点) 字段：录入表单加「知识点」输入框 + datalist（候选来自 KP_BANK 的 kp 部分），必填；错题本卡片显示 📌知识点；种子易错题在 seedGrade 时按 GRADE_KP 映射打上 kp。
- 新增内置题库 `KP_BANK`：键为「学科|知识点」，覆盖小学/初中/高中全部种子知识点（65 个知识点，每个 2 道同类新题，共约 130 道），集中在 app.js 顶部常量，可继续扩充。
- 重写 `renderExtendSetup`/`startExtend`：按 kp(缺则 type) 分组，按钮显示「N错题 +M新题」；练习 = 自己的错题(同kp) + KP_BANK 同类新题(isBank 标记，卡片带 ✨新题徽章)。
- 练习引擎 `startSession` 支持 isBank 项：答对不入库，答错调用 `addBankWrong`（按题干去重，自动 unshift 入错题本并标记 fromExtend、kp 同知识点、mastery=0），形成练习闭环。
- 验证：jsdom 冒烟测试（window.eval 单次执行避免重复声明）——初中 25 个知识点按钮全部匹配到新题；新题答错后自动入本(25→26 且标记 fromExtend)；全部题目含 kp。全部通过。

## 新增：学习小组（微信结合 + 多用户）
- 需求：用户建学习小组（自动生成 6 位编号）→ 他人按编号搜索 → 申请加入 → 组长审批 → 组长可移除成员/转移组长/删除小组；组内成员积分排名、互看错题集（隐私开关：默认仅公开统计概览，开启后才共享完整错题）。
- 现实约束：多用户共享 + 真微信登录必须后端。交付改为「Node+Express 后端 + 同页前端」，运行方式从双击变成 `node server.js` → 打开 http://localhost:3000（已在项目装 express）。
- 后端 `server.js`：Express 托管静态文件 + JSON 文件存储(server_data.json，原子写)；会话用 cookie(sid)；接口：
  - 登录：`/api/auth/me`(取当前用户)、`/api/auth/dev`(体验登录，填昵称即可，无需微信资质)、`/api/auth/logout`；`/api/auth/wechat/start` + `/api/auth/wechat/callback`(OAuth 框架，设 WECHAT_APPID/WECHAT_SECRET/PUBLIC_BASE 环境变量即启用真微信登录)。
  - 小组：`POST /api/groups`(建组，自动 6 位编号)、`GET /api/groups/mine`(我的组)、`GET /api/groups/search?q=编号`(按编号搜索)、`POST /api/groups/:n/join`(申请)、`POST /api/groups/:n/approve`(审批，仅组长)、`POST /api/groups/:n/reject`、`POST /api/groups/:n/transfer`(转移组长，仅组长)、`DELETE /api/groups/:n`(删除，仅组长)、`GET /api/groups/:n/members`(成员+积分排行，按 points 降序)。
  - 资料共享：`POST /api/profile`(同步 name/points/public/questions 快照到服务端)、`GET /api/profile/:id`(按对方 public 返回：公开给统计+完整错题，否则仅统计概览且 questions=[])。
  - 所有受保护接口用 requireUser 校验 401；组长操作校验 leaderId 返回 403。
- 前端改造：
  - 登录门 `#login-overlay`：微信按钮(指向 /api/auth/wechat/start) + 体验登录(填昵称 → /api/auth/dev)；未登录锁全站；api() 封装带 cookie 的 fetch 并 401 时弹登录门。
  - 侧栏新增「学习小组」视图 `#view-groups`：我的小组(创建/进入)、按编号搜索、申请；进入小组看成员排行+「查看错题」(隐私控制)；组长见审批/移除/转移/删除面板。
  - 错题/积分变更后 `save()` 自动调用 `syncProfile()` 把积分+公开设置+错题快照同步到服务端；设置页加「公开我的错题」开关 `#set-share` + 「退出登录」。
  - 显示用 currentUser 全局变量；关注/排名按 Group.members 映射 profile 快照取 points。
